## 1. 我们是谁

本应用由上述运营主体提供，帮助中文和俄语使用者在独立翻译会议中进行按住说话的语音识别、翻译、译文语音播放和会议记录管理。本政策说明我们如何收集、使用、存储、共享和保护个人信息，以及您如何行使权利。

## 2. 我们处理的信息

### 2.1 账号与身份信息

当您注册或登录时，我们可能处理显示名称、手机号、邮箱、密码哈希、验证码状态、头像、账号状态，以及设备会话所需的设备标识、平台、Token 哈希、最后活跃时间和撤销时间。App 不保存您的明文密码。

访客加入时，我们处理您主动填写的姓名或显示名称、所属公司、接收会议纪要的邮箱、使用语言、当前设备标识和访客身份有效期。上述会议身份资料完整后才能进入会议和发送语音；访客身份仅用于受邀的当前会议。

### 2.2 客户与会议信息

主持人可以创建客户资料，包括显示名称、公司、国家或地区、手机号、邮箱和备注。主持人应只填写其有权处理的信息，并向相关人员说明用途。

会议记录可能包含会议主题、主持人、客户、参与者、加入/离开时间、会议状态、历史访问期限和消息数量。每个会议只绑定一个客户；不同客户和会议的数据不会因界面操作自动合并。

### 2.3 语音和翻译内容

只有在您主动按住说话时，App 才录制当前语音片段。该片段发送到服务端，用于：

1. 识别中文或俄语原文；
2. 生成目标语言译文；
3. 对最终译文生成语音；
4. 把最终结果同步给当前会议的授权参与者。

当前版本默认保存最终原文、最终译文、participantId、发言时的姓名/公司/语言快照、时间、顺序、翻译状态和必要的技术请求标识。我们默认不长期保存完整原始录音、不保存临时识别结果、不保存静音或无效音频。当前服务端在单次请求内以内存缓冲处理上传，不创建长期原始录音文件；请求完成或失败后释放该缓冲。

为方便重播，最终译文的合成音频会保存在私有存储中；正式保留期限为 `[填写期限]`。上线前必须让对象存储生命周期/删除流程与该期限一致并完成验收；当前代码会在删除会议或 Host 账号时同步清理引用的合成音频，任一在线对象删除失败时不提交对应数据库删除。该同步路径尚不等于对象存储、Redis、第三方和备份已完成跨系统删除传播。若未来增加原始录音保存，会另行说明并取得单独授权。

### 2.4 设备、权限和日志

为提供功能，我们可能处理 App 版本、操作系统、网络状态、错误类型、请求 ID、内部资源 ID、连接和性能信息。普通运行日志不记录完整语音、完整原文/译文、密码、验证码、Token、房间令牌或 API Key。

App 只在使用相关功能时申请：

- 麦克风：录制您主动提交的语音片段。
- 相机：扫描会议二维码。
- 通知：仅在基础推送启用且您同意时，用于会议邀请或状态提醒；实时翻译不依赖推送。

您可以在系统设置中关闭权限，但相应功能将不可用。

## 3. 使用目的

我们仅为以下目的处理信息：

- 注册、登录、刷新会话、设备管理和账号安全。
- 创建和管理客户、会议、参与者、邀请和历史权限。
- 完成中俄语音识别、翻译、最终译文语音合成和双端同步。
- 展示、搜索、复制、导出和删除有权访问的会议历史。
- 在主持人明确操作后，将单场会议的最终原文、译文、发言人和时间信息发送给阿里云百炼文本模型，生成带来源依据的结构化会议纪要草稿。
- 由会议主持人明确选择收件人后，逐人发送当前会议的会议纪要，并记录供应商受理或失败状态。
- 断线重连、消息补拉、幂等去重、故障排查、防滥用和安全审计。
- 遵守适用法律、处理用户请求和保护用户、运营主体及第三方合法权益。

当前 AI 会议纪要只在用户有权访问且主持人明确选择的单次会议内处理，不使用会议内容生成跨会议用户画像，不默认做客户长期记忆，也不把实时翻译内容用于广告。模型草稿需经服务端来源校验和主持人确认。若未来启用跨会议分析，将在启用前更新告知并完成必要的授权、质量和安全评估。

## 4. 第三方处理方

为完成服务，信息可能由以下类别的受托服务商处理：

| 类别 | 目的 | 可能处理的信息 | 发布前填写 |
| --- | --- | --- | --- |
| 阿里云百炼 | ASR、机器翻译、TTS、单场会议纪要整理 | 当前语音片段、原文/译文、发言人身份快照、语言、术语与发言时间 | 账号主体、地域、产品名称与政策链接 |
| 云数据库/Redis/对象存储 | 业务保存、实时状态、短期 TTS 音频 | 账号、客户、会议、最终文本和技术数据 | 供应商、地域与保留期 |
| Resend（启用邮件分发时） | 发送会议纪要事务邮件 | 收件邮箱、显示名称、会议纪要正文和技术发送标识 | 账号主体、处理地域、保留期与政策链接 |
| APNs/FCM（若启用） | 会议提醒 | 推送 Token 和不含翻译正文的提醒 | 实际启用状态与政策链接 |
| 崩溃/监控服务（若启用） | 稳定性和安全 | 脱敏设备、错误和性能数据 | 服务名、开关与地域 |

我们要求受托方按指示、保密和安全措施处理，不得将信息用于其独立广告目的。正式发布的第三方清单必须与生产代码、SDK、网络请求和合同一致；未启用的服务不应提前声明为正在收集。

## 5. 共享、转让与公开披露

我们不会出售个人信息。除以下情况外，不向无关第三方共享：为实现本政策所述功能委托服务商处理；获得您的单独同意；依法履行义务、响应有权机关；或为保护生命、财产和重大合法权益所必需。

企业合并、分立、收购或资产转让涉及个人信息时，我们会依法告知接收方，并要求其继续受本政策约束或重新取得同意。

我们不会公开披露会议内容，除非取得明确授权或法律另有要求。邀请链接可能被接收人转发；加入仍需身份、房间状态和服务端权限验证。

主持人主动执行“邮件分发会议纪要”时，服务端只向其选择且仍有权限的参会者逐人发送，不使用公开抄送或把其他收件人的邮箱放入邮件。邮件离开本服务后还会由收件人的邮件服务商按其政策处理，请勿通过普通邮件发送不必要的密码、Token、原始语音或其他敏感凭证。

## 6. 存储地点和期限

主要存储地域：`[填写国家/地区和云地域]`。如产生跨境传输，应在发布前说明接收方、目的、信息类型、安全措施和用户权利，并完成适用法律要求的评估、合同或同意。

我们仅在实现目的和满足法律要求所需期间保存：

- Access Token 短期有效；Refresh Token 按设备保存并轮换/撤销。
- 账号、客户、会议和最终翻译保存至用户删除、账号注销、主持人设定到期或运营政策期限届满。
- Guest 访问权按 `NO_ACCESS_AFTER_END`、24 小时、7 天或永久策略执行；访问到期不必然等于会议创建者的业务记录立即删除。
- 原始上传音频只存在于单次请求的内存缓冲中，处理结束后释放；TTS 资产按正式期限通过已验收的对象存储生命周期或删除流程清理。
- 会议内邮箱快照和纪要邮件受理状态随会议记录保存；账号注销时清除相关收件邮箱及姓名/公司快照，会议删除时随 conversationId 级联删除。邮件供应商和收件方邮箱中的副本按其政策和适用要求处理。
- 安全审计、备份和依法保留数据按 `[填写期限]` 保存，期限届满后删除或匿名化。

备份中的已删除数据在备份生命周期内受隔离保护，并在恢复时重新应用删除清单。

## 7. 数据隔离与安全

每个会议绑定固定 `ownerId`、`contactId` 和 `conversationId`；每条消息绑定该会议和参与者。服务端验证所有访问权限，房间码不是唯一安全凭证。我们使用短期 Access Token、轮换 Refresh Token、设备撤销、TLS、Secret 服务端存储、最小权限、日志脱敏、限流、审计和备份恢复措施。

互联网服务不存在绝对安全。如果发生可能影响您权益的安全事件，我们会依适用法律采取补救并告知事件性质、可能影响、处置措施和您可采取的建议。

## 8. 您的权利

在适用法律范围内，您可以请求：

- 访问、更正或补充个人资料。
- 获取您有权访问的会议最终文本和基础导出。
- 删除主持人有权管理的会议记录。
- 撤回可选权限或同意；撤回不影响此前处理的合法性。
- 注销账号并删除/匿名化相关信息。
- 获取对自动化处理和信息处理规则的说明。
- 投诉或联系我们处理其他法定权利。

App 内路径：`设置 > 账号与安全 > 注销账号`。详细流程见 [账号注销与数据删除](./ACCOUNT_DELETION.md)。为防止冒名操作，我们可能验证身份；不会要求提供与请求无关的额外信息。

## 9. 儿童和未成年人

本服务面向成年商务交流用户，不主动面向未成年人。若所在地区法律要求监护人同意，未成年人应在监护人阅读并同意后使用。发现未经适当授权处理未成年人信息时，请通过上述联系方式联系我们删除或采取其他措施。
